# 邮箱服务商配置与故障排查

供 email-sender 技能按需查阅。SKILL.md 走通主流程即可；遇到"某邮箱怎么配""发送/登录报错"再读这里。

## 服务商预设速查

`config set --provider <名>` 会自动带出下表的 host/port/tls，用户只需再给邮箱地址和授权码。

| provider | host | port | tls | 登录凭据 |
|---|---|---|---|---|
| 163 | smtp.163.com | 465 | ssl | 客户端授权码 |
| 126 | smtp.126.com | 465 | ssl | 客户端授权码 |
| qq | smtp.qq.com | 465 | ssl | 客户端授权码 |
| exmail | smtp.exmail.qq.com | 465 | ssl | 登录密码或专用密码 |
| aliyun | smtp.aliyun.com | 465 | ssl | 登录密码或授权码 |
| feishu | smtp.feishu.cn | 465 | ssl | 客户端专用密码 |
| gmail | smtp.gmail.com | 465 | ssl | 应用专用密码（需两步验证） |
| outlook | smtp.office365.com | 587 | starttls | 账户密码或应用密码 |

不在表内的邮箱，用 `--host --port --tls` 手动指定；多数国内邮箱是 `465 + ssl`，企业/海外邮箱常见 `587 + starttls`。

## 授权码怎么拿（关键）

绝大多数邮箱的 SMTP 登录用的**不是网页登录密码**，而是单独生成的"客户端授权码/应用专用密码"。这是为了：即使授权码泄露，攻击者也只能发信、拿不到你的账号主控权，且可随时单独吊销。所以引导用户时要明确区分这两者，否则必然报 `535 认证失败`。

- **163 / 126**：网页登录邮箱 → 设置 → POP3/SMTP/IMAP → 开启 IMAP/SMTP 服务（需手机短信验证）→ 系统生成一串授权码。
- **QQ**：设置 → 账户 → POP3/IMAP/SMTP 服务 → 开启 → 按提示短信验证 → 生成授权码。
- **Gmail**：先开启两步验证 → Google 账户 → 安全 → 应用专用密码 → 生成 16 位密码（去掉空格填入）。
- **Outlook/Office365**：一般可用账户密码；若开了两步验证，需在账户安全里生成应用密码。
- **飞书邮箱**：邮箱设置 → 客户端专用密码。
- **腾讯企业邮(exmail)**：可用登录密码；若管理员开启了"仅允许专用密码登录"，则在设置里生成。

## 发送/登录报错排查

脚本报错时对照下表定位；`check --test` 可在不发信的情况下单独验证登录。

| 现象 | 可能原因 | 处理 |
|---|---|---|
| `535 authentication failed` | 用了登录密码而非授权码；授权码填错；未开启 SMTP 服务 | 重新生成授权码，`config set --password` 更新；确认已开 IMAP/SMTP |
| 连接超时 / `TimeoutError` | host/port/tls 不匹配；网络或防火墙拦截 465/587 | 核对预设参数；试另一种 tls（ssl↔starttls 对应换 465↔587） |
| `SMTPServerDisconnected` | 端口与 tls 组合错误（如 587 配了 ssl） | 465 用 ssl、587 用 starttls |
| `553`/`554` 发件人被拒 | from 与登录账号不一致；被判垃圾 | 让 from 等于登录邮箱；避免高频群发、精简可疑链接 |
| 能登录但对方收不到 | 进了垃圾箱；收件地址错 | 让用户查垃圾箱；核对收件人拼写 |
| 群发部分失败 | 收件人过多触发频控 | 分批发送；企业邮注意单日上限 |

## 安全提醒

- 授权码等同密码。配置文件默认写在 `$INNO_HOME/email.json` 并设 600 权限，不要复制到会被提交/分享的文件，也不要在对话里回显。
- 一旦怀疑授权码泄露，去邮箱设置里"重置/删除授权码"即可失效，无需改主密码。
