# 研究方法论：多视角检索式深度研究

这是 storm-research 背后的方法论。它讲的是**为什么这样研究有效**，与任何具体实现、框架、工具无关。
思想源头是斯坦福 OVAL 实验室对自动化研究的研究，但下面的原理独立成立——换任何 agent、任何检索工具都适用。

## 一个核心命题：研究的瓶颈是"问"，不是"写"

大多数人以为做研究的难点在"组织和写作"。其实不是。

把一个主题交给 LLM 直接写，它会给你**主流框架下的平均答案**——最常见的说法、最表面的结论。这不是因为它不会写，而是因为它没问对问题。

真正的研究者（比如博士生）做的是另一件事：他们不问一个问题，他们问五个。从五个不同立场反复追问，逼出表面之下的东西。一份博士级的预调研，靠的是**问题的广度和深度**，而非文笔。

所以这套方法的第一性原理是：**自动化研究 = 自动问出好问题 + 让每个回答都有据可查。**

## 三根支柱（缺一根就退化）

### 支柱一：多视角（Multi-perspective）

同一个主题，不同立场的人看到完全不同的东西：

- 一线实践者知道现实里什么管用、什么是坑
- 怀疑论者专挑逻辑漏洞和被夸大之处
- 追踪资金的视角看清谁出钱、谁获利
- 历史视角认出这是不是旧模式的重演
- 学术视角关心证据强度

单一视角必然有盲区。多视角的价值不只是"更全面"，而是**视角之间的冲突会暴露出单一框架永远看不到的东西**。这是整套方法里信息密度最高的产物来源。

但对立视角有一个代价：它们为"制造冲突"而生，可能吵得很深，却**集体漏掉平淡却必要的基础话题**（比如争得火热时没人问"那到底该怎么用"）。所以在对立视角之外，再加一个**覆盖视角**：不带立场，专做**系统性话题分解**——穷举"这一类主题通常必须讲清的标准话题面"。这是把 content/deep research 最扎实的长处（query decomposition 的机械完整性）吸收进来当兜底。

于是广度有了**两个引擎**：对立视角负责"深层 + 跨框架"的覆盖（别人想不到的角度），覆盖视角负责"机械完整性"（标准话题面一格不漏）。两者的产物进入同一条检索→接地流水线，所以基础话题不是事后补的空架子，而是被真实检索、真实接地的内容。

### 支柱二：检索接地（Retrieval / Grounding）

这是分界线：**有它，是 research；没它，是 persona-prompting。**

如果只让一个模型扮演五个角色、各自凭记忆发言，那五个角色共享同一个模型的同一套盲区和偏见，会满怀自信地一起编造。视角再多也是假的多样性。

唯一的解药是：**每个视角的每个回答，都必须基于真实检索回来的外部信息。** 检索不到，就明确说"无法回答"，绝不用记忆补全。这样多视角才有真实的信息差，冲突才是真冲突。

### 支柱三：可追溯（Citation）

每一个论断都要能指回它的来源。内联引用不是格式要求，而是**让结论可被检验**的机制——读者（和你自己）能顺着链接核实，能判断这句话靠的是权威一手资料还是某条二手解读。

没有可追溯性，再漂亮的报告也只是"听起来很专业"。

## 方法的形状：一条信息精炼流水线

```
视角(Persona) → 提问(Question) → 检索(Retrieve) → 接地作答(Answer)
   → 找矛盾(Contradiction) → 组织(Outline) → 带引用写作(Cite) → 自我评审(Review)
```

每一环都有它存在的理由：

| 环节 | 解决什么问题 |
|---|---|
| 视角发现 | 打破单一框架，制造问题的多样性 |
| 带视角提问 | 把"列问题"升级成"带立场地追问"，逼出深度 |
| 问题→检索 | 把抽象问题变成可搜索的查询，去拿真实证据 |
| 接地作答 | 强制每句话有据可查，宁可拒答不可编造 |
| 矛盾图 | 把多视角的信息差提炼成"分歧/共识/缺口"——理解的精华 |
| 两步大纲 | 先有骨架再注入深度，避免信息驱动的零散结构 |
| 带引用写作 | 落实可追溯性 |
| 自我评审 | 补上"不会自我批判"这个固有弱点 |

## 必须诚实标注的三个局限

这套方法不是万能的，用它的人要清楚边界：

1. **不会自我批判**。来源偏见、把某条信息错配到某个论断上——这些会悄悄混进去。所以流程末尾**强制**加一道自我同行评审，专门揪强/弱论点、偏见、缺失角度。

2. **偏向主流**。本质是综合"网上已有的、可检索到的主流说法"。对冷门、前沿、需要非共识洞见的主题，容易流于"显而易见和平均水平"。遇到这类题目，必须在产出里明确提示"本报告偏向主流框架，可能遗漏非常规视角"。

3. **不是发表级成品**。定位是**写作前的预调研**——给你一个有来源、有结构、覆盖多视角的高质量起点，而不是可以直接交付的终稿。重大决策仍需结合一手调研和人的判断。

## 一句话记住

**多视角制造广度，检索接地保证真实，强制引用守住可追溯——三者合一，才把 LLM 从"凭记忆答"变成"带证据的研究"。** 任何一根支柱倒了，它就退回成一个听起来聪明、但说不清来源的内容生成器。
