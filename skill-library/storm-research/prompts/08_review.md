# Step 8 — 自我同行评审 (Self Peer-Review)

> 作用：补上"不会自我批判"这个固有弱点，让模型给自己的成果当审稿人。

## 为什么需要这一步

这套方法有一个固有弱点：**流程本身不会自我批判，来源偏见和事实错配会悄悄混进去**。这一步专门补上，让模型给自己的成果当审稿人。

## 8a. 同行评审

```
主题：{TOPIC}
你刚完成的研究报告 + 引用表：
{完整草稿}

以严苛审稿人的身份评审你自己的报告：

1. 强论点：哪些结论来源充分、多方印证、可信度高？
2. 弱论点：哪些结论只靠单一来源、或来源可靠性存疑、或是推断而非事实？
3. 潜在偏见：报告整体偏向哪种框架？来源是否集中在某一类立场？
4. 事实错配风险：哪些 [n] 引用可能对不上它支撑的论断？逐一核对可疑处。
5. 缺失角度：Step 5 矛盾图里的盲区，报告补上了吗？还漏了什么？
6. 可靠性评分：给整篇报告一个 1-10 的可靠性评分，并说明依据。
```

## 8b. 去重清理

```
你是忠实的文本编辑。找出文章里重复的信息并删除，确保不重复。
不删除任何非重复内容。保留所有行内引用和文章结构（"#" "##" 等层级）。
```

## 主流偏向提示（重要）

这套方法本质是综合"网上已有的、可检索到的主流说法"。若本次主题冷门或需要非共识洞见，在自查报告里明确写一句：

> ⚠️ 本报告基于可检索的主流来源综合而成，可能偏向主流框架，遗漏非常规或前沿视角。重大决策请结合一手调研。

## 产出

把 8a 的评审作为报告的 `## 自查报告` 章节，把 8b 清理后的正文作为最终报告主体。
