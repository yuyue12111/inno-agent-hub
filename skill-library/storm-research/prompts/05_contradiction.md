# Step 5 — 矛盾图 / 缺口发现 (Contradiction Map)

> 作用：把多视角的信息差提炼成"分歧 / 共识 / 缺口"——这是整个流程里信息密度最高的产物。

## 内核

在多视角的研究材料里，找出各视角"打架"的地方。分歧处才是真正理解的所在。

## 提示词模板

```
主题：{TOPIC}
各视角已收集的带来源回答：
{所有 persona 的 Step 4 输出 + 引用表}

请产出一张矛盾图：

1. 分歧点：哪些问题上不同视角给出冲突的结论？分别是什么立场，各自的来源是什么？
   - 标注每一方依据的 [n] 来源，以及来源的可靠性。
2. 共识点：哪些结论 5 个视角都认同？（大概率为真，但仍标注来源）
3. 盲区/缺口：哪些重要角度没有任何视角触及？没人提到的地方往往是整个领域的机会或风险。

对每个分歧点，简述"为什么会有这种分歧"——是数据不同、时间不同、激励不同，还是定义不同。
```

## 为什么这样设计

多数人跳过这一步，而它正是"表面理解"与"真正专业"的分水岭。它是综合写作前的关键中间产物，直接进入最终报告的 `## 矛盾分析` 章节。

判定规则：
- 若所有视角全部一致 → 大概率为真
- 若某重要角度无人触及 → 你找到了领域缺口
