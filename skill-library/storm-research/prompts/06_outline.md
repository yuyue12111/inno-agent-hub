# Step 6 — 两步法大纲 (Two-Stage Outline)

> 作用：先有结构骨架再注入采集到的深度，避免信息驱动的零散大纲。

## 内核

先凭已知写一版草稿大纲，再用采集到的信息精修成更翔实的大纲。两步走。

## 6a. 草稿大纲（凭已知）

```
要写的主题：{TOPIC}

为该主题写一份文章大纲。格式：
- 用 "#" 表示一级章节，"##" 表示二级，"###" 表示三级
- 不要包含主题名本身
- 只写大纲，不写其他内容
```

## 6b. 精修大纲（用采集信息）

```
要写的主题：{TOPIC}
当前草稿大纲：
{6a 的输出}
信息采集对话/矛盾图：
{Step 4 各视角回答 + Step 5 矛盾图}

基于采集到的信息改进上面的大纲，让它更翔实、更能覆盖多视角和分歧点。
同样用 "#" "##" "###" 标层级，不含主题名本身。
```

## 6c. 话题完整性核对（兜底）

精修大纲后，做一次机械完整性核对，防止对立视角吵得很深却漏掉平淡必要的基础话题：

```
对照 Step 2 覆盖视角产出的"标准话题清单"，逐条核对当前大纲：
- 每个标准话题面（定义/机制/证据/安全/剂量用法/同类对比/监管……）是否都有章节或子节承载？
- 若某个标准话题在大纲里缺席：
  · 已有检索数据支撑 → 直接补进大纲
  · 无检索数据 → 标记为"缺口"，回到 Step 3 补检索，或在自查报告里列为缺失角度
- 禁止用模型记忆把缺席话题"脑补"成一段。
```

## 规则

- 不要单独的 introduction / conclusion 章节（最终由摘要段统领）。
- 让矛盾图里的关键分歧在大纲中有对应章节或子节。
- 大纲的章节标题后面会被当作检索查询用（Step 7），所以标题要具体、可检索。
- **完整性 + 深度都要**：既要矛盾图的分歧点有章节（深度），也要覆盖视角的标准话题面无缺席（完整性）。

## 为什么这样设计

两步法的价值：草稿大纲提供结构骨架（防止信息驱动的大纲零散），精修步骤注入真实采集的深度。两版都保留，写报告时可在自查环节对照"草稿覆盖了什么、精修补了什么"。

6c 的完整性核对是把 content 调研的"机械完整性"落到最后一道关：对立视角保证了深层覆盖，这道核对保证了常规话题面不被漏掉，两种广度在大纲层合流。
