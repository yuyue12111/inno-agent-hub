# Step 2 — 带视角提问 (Perspective-Guided Question Asking)

> 作用：把"列问题"升级成"带立场地追问"，用上一轮回答驱动下一轮，逼出深度。

## 内核

你是一位带特定关注点的研究者，正在向专家提问以获取撰写该主题所需的信息。问出好问题。

## 提示词模板

```
主题：{TOPIC}
你的视角（除了研究者身份之外的特定关注点）：{PERSONA}
已问过的问题与回答：
{CONVERSATION_HISTORY}

规则：
- 一次只问一个问题
- 不要问已经问过的问题
- 问题必须扣紧主题，且体现你这个视角独有的关切
- 追问要往深处走：上一轮的回答暴露了什么值得继续挖的点？
- 当你没有更多问题可问时，输出："谢谢，我了解清楚了。"

为每个视角生成 2-3 轮这样的问题。
```

## 覆盖视角的特殊模式：话题分解（query decomposition）

上面的对立视角靠"好奇 + 追问"驱动。但 Step 1 里那个**覆盖视角**不一样——它靠**系统分解**驱动，专门保证机械完整性。对它用下面这个模板，而不是上面的追问模板：

```
主题：{TOPIC}

你是覆盖视角，不带立场。任务：穷举"这一类主题通常必须讲清的标准话题面"。
先判断本主题属于哪一类（如：某化合物/产品评测、某公司分析、某技术介绍、
某政策评估、某现象解释……），再套用该类主题的标准结构骨架，列出所有必答话题。

以"某化合物/保健品"为例，标准话题面通常包括：
定义与分类 / 作用机制 / 发现与历史 / 人体证据 / 动物或体外证据 /
安全性与副作用 / 剂量与用法 / 与同类物的对比 / 生物利用度 / 监管现状 / 市场与产品。

输出：一张"标准话题清单"，每个话题转成 1-2 个可检索的具体问题。
这些问题同样进入 Step 3 检索、Step 4 接地作答——保证基础面被真实检索覆盖，而非到大纲阶段才发现漏了。
```

## 为什么这样设计

核心洞察：**自动化研究的难点是"自动问出好问题"，不是写作**。直接让模型"列问题"很浅。用 persona 约束 + 模拟多轮追问（上一轮答案喂回来再问），逼出深度和广度。

**两种提问机制并存**：对立视角用"追问"挖深度和非常规角度；覆盖视角用"分解"保基础完整性。后者就是把 content 调研的 query decomposition 吸收进来——但关键区别是，它的产物**和对立视角的产物一起进入同一条检索→接地流水线**，所以基础话题不是事后补的架子，而是被真实检索、真实接地的内容。

注意：历史过长时，保留最近几轮的完整问答、更早的只留问题摘要（省 token）。
