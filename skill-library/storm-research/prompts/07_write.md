# Step 7 — 带引用写作 (Grounded Section Writing)

> 作用：把可追溯性落地——按章节写作，每个论断内联引用，指回真实来源。

## 7a. 逐章节写作

```
页面主题：{TOPIC}
要写的章节：{SECTION_NAME}（含其子节大纲）
收集的信息（先用本章标题从全局引用表检索相关条目）：
{相关的 [n] 来源 + snippet}

写这个章节，要求：
1. 用 "#" 标章节标题，"##" "###" 标子节
2. 行内用 [1][2]...[n] 标注引用（例："美国的首都是华盛顿特区[1][3]。"）
3. n 对应全局引用表里的真实 URL，不要编造引用号
4. 不写引言/结论章节，不重复页面标题
5. 每个论断都要落在某个 [n] 上；没有来源支撑的论断不要写
```

## 7b. 摘要引导段（写在报告最前）

```
页面主题：{TOPIC}
完整草稿：{所有章节}

写一段引导摘要：
1. 作为全文的简明概述，能独立成段
2. 点明主题、确立背景、说明为何重要、概括最重要的点（含主要争议）
3. 不超过 4 段，恰当处加 [n] 引用
```

## 规则

- 每章先做"标题 → 检索引用表"的映射，只用相关来源，避免无关信息稀释。
- 内联引用号必须对得上 Step 3 的全局引用表。写完后核对一遍：报告里出现的每个 [n] 都能在参考来源列表找到。
- 多个章节可分别独立写，最后拼装。

## 为什么这样设计

强制内联引用是可追溯性的落地。刻意跳过 intro/conclusion 独立章节，改由摘要段统领——避免重复和空洞的总结。
