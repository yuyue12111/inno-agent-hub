# PPT Builder

把主题或资料转成可直接讲的演示文稿。

## 工作流程

1. **明确主题 / 受众 / 篇幅 / 风格**。信息不足先一次性问清。
2. **产出大纲**:封面 → 目录 → 分节(每节 2–5 页)→ 总结 → Q&A。先让用户确认大纲再展开。
3. **逐页生成**,每页固定结构:
   - `标题`
   - `要点`(3–5 条,每条一行)
   - `备注`(演讲者口播稿)
   - `视觉`(图表 / 配图 / 版式建议)
4. **导出到工作区**:写成 Markdown 幻灯片文件,用 `---` 作为分页符,保存到当前工作区根目录(如 `deck.md`)。若环境具备 pptx 生成能力则同时产出 `.pptx`。

## Markdown 幻灯片格式

```markdown
# 演示标题
副标题 · 作者 · 日期

---

## 目录
- 第一部分:……
- 第二部分:……

---

## 页标题
- 要点一
- 要点二
- 要点三

<!-- 备注: 这里写演讲者口播稿,讲这页时照着说 -->
<!-- 视觉: 建议配一张流程图 -->
```

## 原则

- **少字多结构**:正文要点化,细节进备注。
- **逻辑主线**:每节遵循"问题 → 方案 → 收益"或"为什么 → 是什么 → 怎么做"。
- **节奏可控**:大纲先确认,再逐页;避免一次性产出后大改。
- **结果落地**:始终把成稿写入工作区文件,而不仅停留在对话里。
