# 课堂互动网页构建

当用户要为某节课制作课堂互动网页时,按本技能产出。

## 产出形式

- **单个 `index.html` 文件**,CSS 与 JS 全部内联,不引用任何外网资源(CDN、字体、图片链接一律内嵌或省略),保证教室断网也能双击打开。
- **适配教室大屏**:基础字号大(正文 ≥ 20px,标题更大),高对比配色,按钮和可点区域足够大,适合远距离观看与触屏 / 鼠标操控。

## 页面结构(默认四个模块,可按需增减)

1. **课程知识点展示区**:本节课要点,用卡片或列表呈现,支持逐条点击展开 / 收起。
2. **随堂互动答题模块**:若干选择 / 判断题,学生选择后即时高亮对错并给出简短解析,可显示得分。
3. **实时课堂问答互动区**:展示讨论问题,提供随机点名、抢答计时、倒计时等课堂操控小工具(纯前端实现)。
4. **课程小结展示模块**:本节课收束要点,呼应知识点展示区。

## 实现约定

- 用顶部导航或分区锚点在四个模块间切换,方便教师在课堂上跳转。
- 交互用原生 JS 实现,状态保存在内存即可(刷新重置),无需后端。
- 题目与知识点内容,优先取用工作区里已有的教案 / 讲解稿;没有就根据用户给的课题生成。
- 视觉风格(排版 / 配色 / 动效)遵循本工作区的 `frontend-design` 技能的设计指南,挑定一个明确美学方向,避免 AI 套路感;同时保持上面"大屏可读、按钮好点"的课堂约束。

## 工作流程

1. 确认年级、学科、课题与本节课知识点(信息不足先问清)。
2. 先用一段话向用户说明网页将包含哪些模块与交互,确认后再生成。
3. 生成完整 `index.html` 并写入当前工作区根目录。
4. 告诉用户文件位置,以及"双击在浏览器打开 → 投到大屏"的使用方式。
