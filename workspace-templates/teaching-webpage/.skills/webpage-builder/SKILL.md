# 课堂互动网页构建

当用户要为某节课制作课堂互动网页时,按本技能产出。

## 产出形式

- **单个 `index.html` 文件**,CSS 与 JS 全部内联,不引用任何外网资源(CDN、字体、图片链接一律内嵌或省略),保证教室断网也能双击打开。
- **适配教室大屏**:基础字号大(正文 ≥ 20px,标题更大),高对比配色,按钮和可点区域足够大,适合远距离观看与触屏 / 鼠标操控。

## 页面结构(默认四个模块,可按需增减)

1. **课程知识点展示区**:本节课要点,用卡片或列表呈现,支持逐条点击展开 / 收起。
2. **随堂互动答题模块**:若干选择 / 判断题,学生选择后即时高亮对错并给出简短解析,可显示得分。
3. **实时课堂问答互动区**:展示讨论问题,提供随机点名、抢答计时、倒计时等课堂操控小工具(纯前端实现)。
4. **课程小结展示模块**:本节课收束要点,呼应知识点展示区。

## 实现约定

- 用顶部导航或分区锚点在四个模块间切换,方便教师在课堂上跳转。
- 交互用原生 JS 实现,状态保存在内存即可(刷新重置),无需后端。
- 题目与知识点内容,优先取用工作区里已有的教案 / 讲解稿;没有就根据用户给的课题生成。
- 视觉风格(排版 / 配色 / 动效)遵循本工作区的 `claude-design` 技能:挑定一个明确美学方向、给出变体、避免 AI 套路感;同时保持上面"大屏可读、按钮好点"的课堂约束。
- 结构化版式(幻灯片 / 表格 / 流程图 / 分区导航等)可参考本工作区的 `visual-explainer` 技能的模板与 CSS 模式。
- **离线优先(教室硬约束)**:`claude-design` 的 React/Babel 模式会从 unpkg 拉 CDN、`visual-explainer` 的 Mermaid 走 jsdelivr CDN——课堂网页**默认不要用这些联网路径**,改用纯 CSS / HTML / SVG;确实需要某个库时,把它的代码**内联**进单文件,保证断网可开。

## 交互与动画(有目的,不装饰)

交互和动画要**服务于理解**,不是为了好看。判据:**去掉它,学生会不会更难懂?会→留,不会→删。**

**鼓励(直接帮助学习):**
- **可操纵**:让学生改参数 / 拖动 / 选择,即时看到结果,做中学。
- **表现变化与过程**:内容涉及运动、变换、步骤、因果时,用动画展示其发生过程。
- **信号引导**:高亮 / 聚焦当前正在讲的元素,把注意力带过去。
- **即时反馈**:答题对错、点击揭示答案,给清晰的视觉反馈。

**避免(增加认知负荷、分散注意):**
- 装饰性动画:入场特效、粒子背景、无意义过渡、纯炫技。
- 不可控的自动连播:动画必须能**单步 / 暂停 / 重放**,由教师或学生控速,而不是自己跑完。

**课堂约束:**
- 大屏可读:动画幅度大、远处看得清,不做细微的 micro-interaction。
- 纯 CSS / JS 实现即可(契合离线单文件);弱机慎用重 WebGL。

## 工作流程

1. 确认年级、学科、课题与本节课知识点(信息不足先问清)。
2. 用 `ask_user_question` 给用户一个**可多选的模块清单**(知识点展示 / 可交互动态演示 / 随堂答题 / 课程小结 / 实时课堂问答 / 自定义),让用户勾选要哪些;可给推荐项,但以用户选择为准。只生成选中的模块。
3. 生成完整 `index.html` 并写入当前工作区根目录。
4. 告诉用户文件位置,以及"双击在浏览器打开 → 投到大屏"的使用方式。
