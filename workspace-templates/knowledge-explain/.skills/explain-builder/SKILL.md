# 知识讲解构建

当用户要梳理某节课的知识点讲解时,按本技能产出。

## 产出形式(两部分,可写入一个或两个 Markdown 文件)

1. **讲解稿**:可直接照着讲的口语化讲解,按知识点顺序展开。
2. **课件大纲**:每页一个要点的提纲,供转成 PPT / 课件。

## 内容要求

- **贴合学段教材**:精准覆盖本节课知识点,不遗漏、不超纲。
- **通俗易懂**:规避晦涩术语,先讲直觉与例子,再上定义。
- **分层标注**,每个知识点打标签:
  - 基础知识点 / 重点 / 难点 / 易错点
- **每个关键点配**:一个贴近生活的通俗案例 + 一句话通俗解释。

## 工作流程

1. 确认年级、学科、课题(信息不足先问清)。
2. 先列出知识点清单并标好分层,确认后再展开讲解稿与课件大纲。
3. 写入当前工作区根目录,并告知文件位置。
