# 教学目标设计（知识与规范）

素养导向教学目标的设计要点。素养受控词、标签写法、四基四能口径统一以
`../k12-core-competency/SKILL.md` 为准。

## 一、一条好目标的四要素

可测量目标 = **行为主体 + 条件/支架 + 行为动词 + 表现标准**。
例："学生能在模板支持下，独立求出两方案费用相等的临界点，准确率达 80% 以上。"
- 行为动词要可观察（写出/画出/求出/解释/判断），避免"理解/掌握/体会"这类不可测词。
- 标准尽量量化（准确率、达成率、数量）。

## 二、素养统领与标签

- 目标由核心素养「三会/9 表现」统领，一节课聚焦少数几个素养表现，不铺满。
- **每条目标末尾带素养标签**，格式 `（学科-素养表现）`；一条 ≤2 个标签。
- 主攻的目标要对准学情诊断出的**最薄弱素养**。

## 三、分层

面向不同起点给差异化目标（如 待提升 / 中层 / 优层）：
- 待提升：降低条件难度、加支架（填空式、模板、提示表）。
- 中层：常规条件下独立达成核心标准。
- 优层：迁移/变式/向同伴讲解。
分层目标之间应有明显的条件或标准梯度。

## 四、「教学情境与目标」四字段（常规课口径）

| 字段 | 写法 |
|---|---|
| 核心问题 | 一个用于导入、贯穿例题的问题（**不是长期项目的驱动问题**） |
| 教学情境 | 简短、贴近生活的应用背景，作为例题/应用载体 |
| 学科核心概念 | 本课要建立的核心概念（如「y=kx+b 刻画均匀变化」） |
| 本课学习成果 | 本课结束学生能达到/产出什么（可用于当堂检测） |

> ⚠ 常规课口径：情境是例题载体，**不要**包装成角色扮演 / 项目 / 成果发布。

## 五、四基 / 四能落点

显式标注本课在四基（知识/技能/思想/活动经验）与四能（发现/提出/分析/解决）上的落点，
并让每个落点呼应学情薄弱点。素养是里子，四基四能是抓手。

## 六、教-学-评一致

目标要「可被评价」：写目标时就想清楚它将由哪种评价证据、哪一档量规来检验（等级三=独立规范达成）。
目标 → 评价 → 活动三者对齐。

## 七、一份目标文档的推荐结构

1. 教学情境与目标（四字段表）
2. 分层教学目标（每条带素养标签、含可测标准）
3. 四基/四能落点（表，呼应学情）
4. 重难点（难点通常对应最薄弱处）

## 八、质量准则

- 每条目标可测、分层、带素养标签。
- 主攻目标对准最薄弱素养。
- 常规课口径，无项目/角色包装。
- 每条目标能回指到学情薄弱项或课标要求。
