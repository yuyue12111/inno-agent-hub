# 学情分析 · 数据驱动的班级起点诊断

从班级作答数据出发，看清学生真实起点，为后续目标/内容/活动/评价提供输入。
素养口径以 `../k12-core-competency/SKILL.md` 的 9 个受控词为准。

## 一、输入数据

优先接受**明细长表**（每行=一个学生在一道题上的作答），关键列：
`student_id, question_id, knowledge_point, competency, difficulty, full_score, score, is_correct, error_type`。
- 若只有宽表 + 题目表，先在内存里 join 成长表再统计。
- 找不到数据文件时，先问用户数据在哪、用哪一份。

## 二、必做的六项统计

1. **整体分布**：每个学生总得分率 = Σscore / Σfull_score；给出均值、最高/最低、直方分布。
2. **学生分层**：按总得分率分「优 / 中 / 待提升」（默认 ≥80% / 55–80% / <55%，可按数据调整并说明口径），给出各层人数。
3. **分知识点正确率**：按 `knowledge_point` 聚合「掌握率(is_correct)」与「得分率」，排序找最弱知识点。
4. **核心素养薄弱项**：按 `competency` 聚合得分率，**升序**列出，点名最薄弱的 2–3 项（受控词见基座）。
5. **典型错误**：统计 `error_type` 分布，找出高频错因。
6. **难度分层表现**：按 `difficulty`（易/中/难）聚合得分率，看难度爬升处的断层。

## 三、产出：学情分析看板（自包含 HTML）

**单文件、自包含**（图表库用 CDN，如 Chart.js / ECharts；或内联 SVG），适合录屏与浏览器预览。结构建议：

1. 顶部结论条：一句话点出「本班最需要攻克的 1–2 个知识点 + 1–2 个核心素养」。
2. 图表看板（每项统计配一张图）：
   - 学生分层：环形/柱状占比
   - 分知识点掌握率：横向条形（升序，最弱在上）
   - 核心素养得分率：雷达图或条形
   - 典型错误：饼图/条形
   - 难度分层：柱状
3. 关键结论文字：分层特征、薄弱知识点、薄弱素养、主要错因，各 1–2 句。
4. **教学启示**：明确写「后续目标应主攻 X 知识点 / Y 素养」——这是交给教学目标阶段的输入。

## 四、质量准则

- 结论必须**数据驱动**：每个判断都能追溯到某项统计，不凭空贴标签。
- 薄弱项要区分「个别差生」与「全班普遍」（看是否连高分层也失分）。
- 可视化优先、要点清晰；图表配色克制，不堆纯文字。
- 隐私：只用化名/编号，不输出真实个人信息。

## 五、自检清单

- [ ] 六项统计齐全，口径写清。
- [ ] HTML 单文件可直接打开，图表能渲染。- [ ] 顶部有一句话核心结论，底部有「教学启示」。
- [ ] 薄弱素养用基座受控词表述。
