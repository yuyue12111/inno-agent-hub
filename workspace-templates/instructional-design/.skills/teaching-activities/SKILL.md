# 教学活动设计 · 常规新授课的课堂环节

设计一节**常态课**上真正发生的教学活动，产出一份**教学活动设计文档**。
目标/素养口径以 `../k12-core-competency/SKILL.md` 为准，对齐阶段二目标与阶段一薄弱项。

## ⚠ 边界：这是常规课，不是项目式学习（PBL）

- ❌ 不做角色扮演 / 人设（如「你是××参谋」）。
- ❌ 不设「任务段位 / 关卡 / 成果发布会 / 项目产品」。
- ❌ 不硬套 AI 项目组件。
- ✅ 可用一个贯穿的**真实情境**当例题/应用背景，但它只是应用载体，不做成项目。
- ✅ 节奏是「教师精讲、学生多练、当堂反馈」的常态课。

## 一、教学环节（一节课主结构）

```
情境导入 → 新知探究 → 例题讲解/建构 → 巩固练习 → 当堂检测/小结
```
可按课题微调，但保持「一节正常课」的骨架；总时长对齐课时（如 45′）。

## 二、每个活动要标注的字段

用表格逐环节列出，每个活动含：
- **教学活动**（师做什么、生做什么）
- **组织形式**：全班 / 小组 / 个人
- **对应教学目标 / 核心素养**
- **时长**
- **师生分工**、**所需材料/教具**

## 三、必须嵌入「即时反馈」

在关键环节标注 **★ 即时反馈点**：提问、当堂练习抽查、手势应答、错例点评等，
让教师能当堂判断掌握情况并调整——直接呼应学情薄弱项。

## 四、配套模块

- **分层关照**：对待提升 / 中层 / 优层给出不同支架或拔高（呼应学情分层）。
- **认知主线 / 板书结构**：一行画出本课主线。
- **与学情薄弱项的呼应表**：薄弱项 → 对应环节。

## 五、自检清单

- [ ] 是常规课结构，无角色扮演 / 段位 / 发布会 / 项目产品。
- [ ] 每个活动字段齐全（目标素养、时长、组织、分工、材料）。
- [ ] 至少 2–3 个即时反馈点。
- [ ] 有分层关照与学情呼应表。
- [ ] 总时长对齐课时。
