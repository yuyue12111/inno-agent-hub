# 教案构建

当用户要为某节课生成教案时,按本技能产出一份可直接落地的 Markdown 教案。

## 固定结构(按此顺序输出)

1. **课程信息**:年级 / 学段、学科、课题、课时(默认约 40 分钟)
2. **学情分析**:已有基础、认知特点、常见障碍
3. **教学目标**:知识目标 / 能力目标 / 核心素养目标(三维)
4. **教学重点与难点**:分别列出 + 突破策略
5. **教学方法**:并说明选用理由
6. **教学准备**:课件、教具、学具、资料
7. **教学流程**(用表格,逐环节给出时长 / 教师活动 / 学生活动 / 设计意图):

   | 环节 | 时长 | 教师活动 | 学生活动 | 设计意图 |
   |------|------|----------|----------|----------|

   环节顺序:导入 → 新知讲解 → 课堂互动 → 巩固练习 → 课堂小结
8. **板书设计**:核心概念的结构化框架
9. **分层作业**:基础 / 提升 / 拓展,各 2–3 题
10. **教学反思**:留空,供课后填写

## 实现约定

- 内容贴合该学段课标与教材,不超纲。
- 各环节时长加总贴近一课时(默认约 40 分钟)。
- 写成单个 Markdown 文件存入当前工作区根目录,文件名含课题与年级。

## 工作流程

1. 确认年级、学科、课题、课时(信息不足先问清)。
2. 按上面的固定结构产出完整教案。
3. 写入工作区并告诉用户文件位置。
