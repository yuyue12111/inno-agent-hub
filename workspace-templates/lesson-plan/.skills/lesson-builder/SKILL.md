# 教案构建

当用户要为某节课生成教案时,按本技能产出一份可直接落地、且符合循证教学原则的 Markdown 教案。

## 设计方法:逆向设计(先想清楚,再排活动)

构思时按"逆向"顺序思考(产出时再按下面的教师友好结构呈现):

1. **先定目标**:这节课学完,学生应该能*做*什么?
2. **再定证据**:用什么任务 / 题目能证明目标达成?(当堂检测、提问、练习)
3. **最后排活动**:设计能通向目标与证据的教学环节。

三者必须**对齐**:目标 ↔ 教学活动 ↔ 检测 ↔ 作业,不出现"教了没测、测了没教"。

## 教学目标写法(可测量)

- 用**布鲁姆认知层级**的行为动词,由低到高:记忆 / 理解 / 应用 / 分析 / 评价 / 创造(例:"能*说出*…""能*应用*…求解…""能*比较*…")。避免"了解 / 掌握"这类不可观测的词。
- 参考 **SMART**:具体、可测、可达成、贴合课标、限定在本课时内完成。
- 仍按知识 / 能力 / 素养三维组织,但每一条都落到**可观察的行为**。

## 固定结构(按此顺序输出)

1. **课程信息**:年级 / 学段、学科、课题、课时(默认约 40 分钟)
2. **学情分析**:已有基础、认知特点、常见障碍 / 典型迷思
3. **教学目标**:三维 + 布鲁姆动词 + 可测量(见上)
4. **教学重点与难点**:分别列出 + 突破策略
5. **评价设计(达成证据)**:逐条目标对应"怎么测"——课堂提问 / 当堂检测 / 练习,标明形成性还是终结性
6. **教学方法**:并说明选用理由
7. **教学准备**:课件、教具、学具、资料
8. **教学流程**(用表格,逐环节给出时长 / 教师活动 / 学生活动 / 设计意图):

   | 环节 | 时长 | 教师活动 | 学生活动 | 设计意图 |
   |------|------|----------|----------|----------|

   - 环节顺序:导入 → 新知讲解 → 课堂互动 → 巩固练习 → 课堂小结
   - **导入处安排"提取练习"**:用 1–2 个问题唤起上节课 / 前置知识(间隔复习),而非只做情境导入
   - 流程中至少安排 **2 处形成性检测**(提问 / 小测 / 举手表决),据此决定是否回讲
9. **板书设计**:核心概念的结构化框架
10. **分层作业**:基础 / 提升 / 拓展,各 2–3 题;**含 1–2 道对往期知识的间隔复习题**
11. **教学反思**:留空,供课后填写(建议反思:目标达成证据如何、哪处迷思仍在)

## 实现约定

- 内容贴合该学段课标与教材,不超纲。
- 各环节时长加总贴近一课时(默认约 40 分钟)。
- **保证对齐**:逐条目标都能在"评价设计"和"教学流程"里找到对应。
- 写成单个 Markdown 文件存入当前工作区根目录,文件名含课题与年级。

## 工作流程

1. 确认年级、学科、课题、课时(信息不足先问清)。
2. 先在心里走一遍逆向设计(目标 → 证据 → 活动),再按上面的固定结构产出完整教案。
3. 写入工作区并告诉用户文件位置。
