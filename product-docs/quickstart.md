## 🤔 Inno Agent 是什么？

Inno Agent 是一个**个人学习助手**，运行在你自己的电脑上，数据完全本地存储。

和普通 AI 聊天工具最大的不同是：**它会记住你**。

> 普通 AI 助手：每次打开都是陌生人，你需要重新介绍自己的背景、学到哪了、哪里还没懂。
>
> Inno Agent：记录你的学习目标、哪些概念已经掌握、哪里还有误解，每次打开都从你上次停下来的地方继续。

它不只是一个聊天框——它是一个有记忆、会主动提醒你复习、能帮你整理知识库的学习伙伴。

## 🚀 5 分钟跑起来

### 第一步：确认 Node.js 已安装

打开终端，输入：

```bash
node --version
```

如果看到 `v20.x.x` 或更高版本，继续下一步。

如果提示找不到命令，先去安装 Node.js（推荐 LTS 版本，从 nodejs.org 下载）。

> **【截图占位】** _此处插入截图——终端输出 `v22.x.x` 的样子_

---

### 第二步：下载项目并安装依赖

```bash
git clone <仓库地址> inno-agent
cd inno-agent
npm install
npm run build
```

等待几分钟，看到 `✓ built in x.xxs` 就说明成功了。

> **【截图占位】** _此处插入截图——`npm run build` 成功完成后的终端，最后一行显示 build 成功_

---

### 第三步：填写 API Key，启动！

```bash
# 复制配置模板
mkdir -p runtime/config runtime/data runtime/skills workspace
cp config.example.json runtime/config/config.json
```

用文本编辑器打开 `runtime/config/config.json`，找到这两行，换成你的信息：

```json
"apiKey": "替换为你的 API Key",
"baseUrl": "替换为你的 API 服务地址"
```

保存文件，然后启动：

```bash
npm run server -- --home ./runtime --workspace ./workspace
```

浏览器打开 `http://localhost:3000`，你就看到 Inno Agent 了 🎉

> **【截图占位】** _此处插入截图——浏览器打开后的 Inno Agent 初始界面_

## 💬 第一次用，从这里开始

启动后，先试着发一条消息。把下面这句话复制到输入框：

```
你好！我想开始学习机器学习，目标是三个月后能独立完成一个 Kaggle 入门项目
```

👉 复制这句话到输入框，然后回车

你会发现右侧的「学习档案」里，出现了刚才你说的学习目标。这就是 Inno Agent 开始「记住你」的第一步。

**再试一个：让它讲一个概念**

```
帮我解释一下梯度下降是什么，用生活中的例子
```

👉 复制这句话到输入框

> **【截图占位】** _此处插入截图——Agent 给出类比解释，右侧学习档案的「梯度下降」掌握度数值有了变化_

## 🧠 它是怎么「记住你」的？

Inno Agent 把关于你的信息分成三层来管理：

> **【截图占位】** _此处插入截图——右侧学习档案面板，展示目标、掌握度数值和偏好设置三个区域_

**🗂️ L1：学习者画像（关于「你是谁」）**

你的学习目标、每个概念的掌握度（0 到 1 之间的数值）、Agent 发现的误解、你偏好的讲解方式……这些信息每次对话前都会自动注入——哪怕你清空了聊天记录，它也还记得你。

**📖 L2：知识 Wiki（关于「你知道什么」）**

你上传的 PDF、课堂笔记、文章，会被自动整理成结构化的 Wiki 页面，还会生成概念之间的关系图谱。Agent 回答问题时可以主动检索这些内容。

**💬 L3：会话历史（关于「我们聊了什么」）**

近期的对话内容，由系统自动管理，无需手动维护。

---

你可以在右侧「学习档案」标签页直接查看和修改这些信息——它不是藏在黑箱里的，你随时可以纠正 Agent 的判断。

## 📚 归档你的学习资料

有论文、课件、笔记想让 Agent 「读」进去？直接拖进去就好。

**操作：**

1. 点击右侧「笔记本」标签页
2. 把 PDF 或文档拖入上传区域
3. 等一会儿，自动生成摘要、提取概念

> **【截图占位】** _此处插入截图——笔记本面板，拖入一篇 PDF 后自动生成的摘要页面和提取出的概念列表_

上传完之后，可以直接问：

```
这篇论文的核心结论是什么？
```

或者：

```
把这篇文章里关于注意力机制的内容和我已有的知识对比一下
```

归档的内容还会自动出现在知识图谱里，你可以可视化地看到概念之间的关联。

> **【截图占位】** _此处插入截图——知识图谱视图，展示多个概念节点和连线，颜色区分不同类型_

## ⏰ 让它主动找你（定时任务）

学习最怕的就是「学完就忘」。可以让 Agent 定时提醒你复习，或者每天自动生成学习总结。

直接告诉它：

```
帮我设置每天晚上 10 点发送今日学习总结，推送到飞书
```

👉 复制这句话到输入框（前提是已配置飞书，见下一节）

Agent 会自动创建一个定时任务。每晚 10 点，它会整理当天的学习记录，生成报告，直接推送到你的飞书。

> **【截图占位】** _此处插入截图——定时任务面板，展示已创建的「每日学习总结」任务，显示下次运行时间_

也可以在右侧「定时任务」标签页手动创建、管理所有任务。

## 📱 在飞书里直接用（可选）

不想每次都打开浏览器？可以把 Inno Agent 接到飞书，用手机也能随时对话、接收推送。

> **【截图占位】** _此处插入截图——飞书聊天界面，正在和 Inno Agent 机器人对话_

**配置大概分三步（详细步骤见完整文档）：**

1. 在飞书开放平台创建一个自建应用，拿到 App ID 和 App Secret
2. 把这些填入 `config.json`，开启 `channels.feishu.enabled`
3. 重启服务，在飞书里搜索你的机器人名字，发消息

配置好之后，无论是问问题、归档资料，还是接收每日总结推送，全部可以在飞书里完成。

> **⚠️ 注意**：飞书接入需要你的服务有公网地址（或 ngrok 内网穿透）。本地跑的话，公网地址才能接到飞书的事件推送。

## 🙋 常见问题

**Q：启动后访问 `localhost:3000` 是空白页？**

执行 `npm run build` 重新构建一次，然后重启服务。

---

**Q：发消息后一直加载，没有回复？**

90% 是 API Key 或 `baseUrl` 填错了。打开浏览器 F12 → Network → 找 `/api/chat/stream` 请求，看报错信息。

另外注意 `api` 字段：用 Anthropic 系（InnoSpark / Claude 官方）填 `anthropic-messages`，用 OpenAI 系填 `openai-completions`，填错会静默失败。

---

**Q：学习档案里的掌握度是 Agent 自己打的分，感觉不准怎么办？**

在「学习档案」→「知识状态」里直接点击对应概念，手动修改数值就好。也可以直接告诉 Agent：「把梯度下降的掌握度改成 0.3」。

---

**Q：想用本地模型（Ollama）而不是云端 API？**

完全支持。在 `config.json` 里配置：

```json
"baseUrl": "http://localhost:11434/v1",
"api": "openai-completions",
"apiKey": "ollama"
```

推荐 `qwen2.5:14b` 及以上，工具调用能力更稳定。

---

**Q：数据存在哪里？会上传吗？**

所有数据（学习档案、Wiki、对话记录）都存在本地的 `runtime/data/` 目录里，是普通的 JSON 和 Markdown 文件，不会上传任何服务器。
